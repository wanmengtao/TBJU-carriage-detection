# 项目目录结构说明

本文档说明项目统一目录结构与各文件/文件夹的用途。

项目根目录：`Carriage/`（原 `yolo_label/` + `数据集标注/` + `Carriage Number/` 三合一）

---

## 一、`raw_data/` — 原始数据集

存放各任务的图片数据和标注文件，是所有训练数据的源头。

### 1.1 目录总览

| 子目录 | 任务 | 帧数 | 状态 |
|--------|------|------|------|
| `eye_level/` | 车号识别（平视角度） | 152 | 已标注 |
| `side_view/` | 车号识别（侧视角度） | 215 | 已标注 |
| `door_state/` | 车门状态识别 | 258 | 待标注 |
| `track_intrusion/` | 轨道异物侵限检测 | 350 | 待标注 |
| `carriage_rim_debris/` | 车厢沿异物检测 | 350 | 待标注 |
| `debris_materials/` | 异物贴图素材（PNG） | — | 素材库 |

### 1.2 通用结构（以 `eye_level/` 为例）

```
eye_level/
├── train/
│   ├── images/          # 训练集图片（JPG）
│   └── labels/          # 标注文件（Label Studio JSON）
├── val/
│   ├── images/          # 验证集图片
│   └── labels/          # 标注文件
└── test/
    ├── images/          # 测试集图片
    └── labels/          # 标注文件
```

### 1.3 数据集划分比例

所有数据集统一采用 **8:1:1** 划分（train:val:test），随机种子 42。

## 三、训练与测试输出

### 3.1 YOLO 训练产物 (`YOLO_train/runs/`)

每个任务一个子目录，训练完成后自动生成：

```
YOLO_train/runs/
├── README.md                                # 目录说明
├── tbju_detection/                          # 车号检测 (nc=1)
│   └── weights/{best.pt, best.onnx, last.pt}
├── carriage_rim_debris_detection/           # 车厢沿异物 (nc=2)
│   └── weights/{best.pt, best.onnx, last.pt}
├── track_intrusion_detection/               # 轨道异物 (nc=2)
│   └── weights/{best.pt, best.onnx, last.pt}
└── door_state_detection/                    # 车门状态 (nc=1)
    └── weights/{best.pt, best.onnx, last.pt}
```

每个 `{task}_detection/` 内含：weights/（模型）、results.csv（指标）、results.png（曲线）、confusion_matrix.png（混淆矩阵）、inference.py（推理脚本）。详见 `YOLO_train/runs/README.md`。

### 3.2 OCR 训练产物 (`OCR_train/output/`)

```
OCR_train/output/
├── README.md
└── ppocr_rec_carriage_number/
    ├── best_model.pth                       # 最佳模型
    ├── ppocr_rec_tbju.onnx                  # ONNX 模型
    ├── training_log.json                    # 训练日志
    └── evaluation_results.csv               # 评估结果
```

详见 `OCR_train/output/README.md`。

### 3.3 测试产物 (`test_model/results/`)

按任务分类存放，每个任务一个子目录：

```
test_model/results/
├── README.md                                # 目录说明
├── wagon_number/                            # 车号（已完成）
│   ├── test_report.txt, ocr_*.csv, end_to_end_details.csv
│   └── runs/detect/val-N/                   # YOLO 验证可视化
├── carriage_rim_debris/                     # 车厢沿异物（待测试）
├── track_intrusion/                         # 轨道异物（待测试）
└── door_state/                              # 车门状态（待测试）
```

详见 `test_model/results/README.md`。

## 五、数据流

```
raw_data/                           datasets/                       训练
(原始数据+标注)                     (转换产物)                      (模型)
     │                                   ↑                            │
     │ Label Studio JSON                 │                            │
     │ + 原始图片                        │                            │
     │                                   │                            │
     ├──── 车号数据 ────→ scripts/convert_labelstudio_tbju.py ←──────┤
     │                                   │                            │
     │                                   ↓                            │
     │                              YOLO 检测数据集                   │
     │                              OCR 识别数据集                    │
     │                                   │                            │
     │                                   ↓                            │
     │                              YOLO_train/ → 检测模型            │
     │                              OCR_train/  → 识别模型            │
     │                                   │                            │
     │                                   ↓                            │
     │                              test_model/ → 模型评估            │
     │                                   │                            │
     │                                   ↓                            │
     │                              RK3588 板端部署                    │
     │                                                                │
     ├──── 异物数据 ─────────────────→ scripts/augment_debris.py ─────┤
     │     (track_intrusion,             (异物贴图扩充)               │
     │      carriage_rim_debris)                                      │
     │           │                                                    │
     │           ↓                                                    │
     │      扩充后 JSON → convert → YOLO 训练                         │
     │                                                                │
     └──── 车门数据 ──────→ scripts/convert_labelstudio_new.py ──────┘
           (door_state)         (nc=1, 无扩充, 直接转换)
```

### 数据流总结

1. **车号识别**：`raw_data/{eye_level,side_view}` → `scripts/convert_labelstudio_tbju.py` → `datasets/output/` + `datasets/test_output/` → YOLO + OCR 训练 → RK3588 部署
2. **异物检测**：`raw_data/{track_intrusion,carriage_rim_debris}` → `scripts/setup_new_datasets.py` 同步图片 → Label Studio 标注 → `scripts/augment_debris.py` 扩充 → `scripts/convert_labelstudio_new.py` 转 YOLO → 训练
3. **车门状态**：`raw_data/door_state` → `scripts/setup_new_datasets.py` 同步图片 → Label Studio 标注 → `scripts/convert_labelstudio_new.py` 转 YOLO (nc=1) → 训练（无异物扩充）
