# TBJU 数据集标注转换流程与结果

## 1. 概述

本文档记录了从 Label Studio JSON 标注文件到 YOLO 检测数据集 + OCR 识别数据集的完整转换流程，包括标注校验、脚本转换和结果验证三个阶段。

**目标：** 为 TBJU/TBCU 车号区域检测与识别模型训练准备数据集，最终部署到 RK3588（ELF2）开发板。

**数据集视角：**
- 平视（Eye Level）
- 侧视（Side Level）

---

## 2. 第一步：JSON 标注校验

### 2.1 校验规则

根据规范文件（`TBJU数据集转换规范.md`），对每个 JSON 标注文件执行以下检查：

| 检查项 | 规则 |
|--------|------|
| 框与文本配对 | `rectanglelabels` 和 `textarea` 通过 `id` 字段关联 |
| 文本标准化 | 去空格、去横杠、去方括号、转大写 |
| 有效格式 | `^(TBJU\|TBCU)\d{7}$`（前缀 + 7 位数字） |
| 无效文本 | 前缀错误、长度不对、含不确定字符 → review_list.csv |
| 缺失文本 | textarea 为空 → review_list.csv（reason: missing_text） |

## 3. 第二步：运行转换脚本

### 3.1 脚本信息

- **脚本文件：** `convert_labelstudio_tbju.py`
- **转换规范：** `TBJU数据集转换规范.md`

### 3.2 执行命令

**平视数据集：**

```bash
python convert_labelstudio_tbju.py \
  --dataset_dirs raw_data/eye_level \
  --output datasets/output \
  --seed 42
```

**侧视数据集：**

```bash
python convert_labelstudio_tbju.py \
  --dataset_dirs raw_data/side_view \
  --output datasets/output \
  --seed 42
```

### 3.3 脚本修复记录

转换过程中发现并修复了一个图片匹配 bug：

**问题：** JSON 中的 `file_upload` 字段和本地图片文件名都带有 UUID 前缀（如 `e666d968-DJI_...`），但 UUID 值不同。`parse_json_annotations` 用 `clean_filename` 去掉 UUID 前缀后得到 `DJI_...`，而本地文件名仍保留各自的 UUID 前缀（`5ce6a246-DJI_...`），导致集合交集匹配失败。

**修复方案：**
1. 在每个标注条目中存储原始 `file_upload` 名称
2. `find_local_image` 优先使用原始 `file_upload` 名进行精确匹配
3. 匹配检查逻辑同步更新，使用 `find_local_image` 替代集合交集
