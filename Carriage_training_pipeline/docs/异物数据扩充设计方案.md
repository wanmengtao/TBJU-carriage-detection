# 异物数据扩充设计方案

## 1. 背景与目标

**问题：** track_intrusion（轨道异物侵限）和 carriage_rim_debris（车厢沿异物）数据集中，绝大多数帧不包含异物，正样本极度稀缺，直接训练效果差。

**目标：** 通过将透明背景的异物素材（水瓶、石头、纸箱等）随机贴入图像的有效区域，批量生成含异物的训练样本，扩充正样本数量。

**前置条件：**
- 已用 Label Studio 标注了有效放置区域（轨道区域 / 车厢沿区域）
- 已收集异物素材 PNG（透明背景）

### 2.2 Label Studio 标注要求

在 Label Studio 中对每张图标注 **放置区域**（矩形框）：

- **类别名：** `track_region`（轨道区域）或 `carriage_rim_region`（车厢沿区域）
- **标注方式：** 用矩形框圈出异物可能出现的区域
  - 轨道：铁轨面 + 两侧道床/路基
  - 车厢沿：车厢顶部边缘区域

**同一张图可以标注多个放置区域**（如左右两条铁轨）。

### 2.3 Label Studio JSON 格式

```json
[
  {
    "id": 1,
    "annotations": [
      {
        "result": [
          {
            "id": "abc123",
            "type": "rectanglelabels",
            "value": {
              "x": 10.5,
              "y": 60.0,
              "width": 80.0,
              "height": 30.0,
              "rotation": 0,
              "rectanglelabels": ["track_region"]
            },
            "from_name": "label",
            "to_name": "image"
          }
        ]
      }
    ],
    "data": {
      "image": "/data/upload/.../DJI_xxx.jpg"
    },
    "file_upload": "DJI_xxx.jpg"
  }
]
```

**关键字段：**
- `rectanglelabels` 中的类别名：`track_region` 或 `carriage_rim_region`
- `x, y, width, height`：百分比坐标（0-100），相对于图像尺寸
- `file_upload`：图片文件名

## 3. 扩充策略

### 3.1 整体流程

```
Label Studio JSON → 解析放置区域
                         ↓
异物素材 PNG → 随机选择 + 随机变换
                         ↓
在放置区域内随机贴图 → 保存扩充图片 + 生成 YOLO 标签
                         ↓
原始数据 + 扩充数据 → 合并训练
```

### 3.2 异物贴图规则

| 参数 | 策略 |
|------|------|
| 位置 | 在放置区域内随机采样（中心点） |
| 大小 | 原始素材按 0.5x ~ 1.5x 随机缩放，确保不超出放置区域 |
| 旋转 | 0° ~ 360° 随机旋转 |
| 透明度 | 保留 PNG 原始 alpha 通道，自然融合 |
| 数量 | 每张图贴 1~3 个异物（可配置） |
| 避免重叠 | 贴图之间 IoU < 0.3（可配置） |

### 3.3 类别定义

扩充后的数据集包含两个类别（在 Label Studio JSON 中）：

- `track_region` / `carriage_rim_region`：放置区域标签（来自原始标注）
- `debris`：异物标签（由扩充脚本自动生成）

后续通过转换脚本统一转为 YOLO 格式时，类别映射为：

```yaml
nc: 2
names: ['region', 'debris']
```

### 3.4 输出格式：Label Studio JSON

扩充脚本输出 **Label Studio JSON**（与原始标注格式一致），保存在各 split 的 `labels/` 目录下。

**输出文件命名：** `augmented_debris_{split}.json`

**JSON 结构示例：**
```json
[
  {
    "id": 10000,
    "annotations": [
      {
        "result": [
          {
            "id": "abc123def0",
            "type": "rectanglelabels",
            "value": {
              "x": 10.5, "y": 60.0, "width": 80.0, "height": 30.0,
              "rotation": 0,
              "rectanglelabels": ["track_region"]
            },
            "from_name": "label",
            "to_name": "image",
            "origin": "manual"
          },
          {
            "id": "xyz789ghi0",
            "type": "rectanglelabels",
            "value": {
              "x": 35.2, "y": 72.1, "width": 5.0, "height": 8.0,
              "rotation": 0,
              "rectanglelabels": ["debris"]
            },
            "from_name": "label",
            "to_name": "image",
            "origin": "manual"
          }

    "data": {"image": "/data/upload/xxx_aug0.jpg"},
    "file_upload": "xxx_aug0.jpg"
  }

```

**后续转换：** 使用 `convert_labelstudio_tbju.py` 将 JSON 转为 YOLO txt 标签。

## 4. 脚本设计

### 4.1 主脚本：`augment_debris.py`

**功能：**
1. 读取 Label Studio JSON，解析放置区域
2. 加载异物素材 PNG
3. 在放置区域内随机贴图
4. 保存扩充图片 + YOLO 标签

**命令行接口：**

```bash
python augment_debris.py \
  --dataset_dir "raw_data/track_intrusion" \
  --debris_dir  "raw_data/debris_materials" \
  --class_name  "track_region" \
  --num_per_image 2 \
  --aug_ratio 0.5 \
  --seed 42
```

**参数说明：**

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--dataset_dir` | 数据集根目录 | 必填 |
| `--debris_dir` | 异物素材目录 | 必填 |
| `--class_name` | Label Studio 中放置区域的类别名 | `track_region` |
| `--num_per_image` | 每张图贴几个异物 | 2 |
| `--aug_ratio` | 对多少比例的图片做扩充 | 0.5 |
| `--max_iou` | 贴图之间最大 IoU | 0.3 |
| `--seed` | 随机种子 | 42 |

### 4.2 输出结构

运行后在原数据集目录下生成：

```
track_intrusion/
├── train/
│   ├── images/
│   │   ├── DJI_xxx_f0001.jpg          # 原始图
│   │   └── DJI_xxx_f0001_aug0.jpg     # 扩充图
│   └── labels/
│       ├── project-xxx.json            # 原始 Label Studio JSON
│       ├── DJI_xxx_f0001.txt           # 原始 YOLO 标签
│       └── DJI_xxx_f0001_aug0.txt      # 扩充 YOLO 标签
├── val/
└── test/
```

### 4.3 核心模块

```
augment_debris.py
├── parse_labelstudio_json(json_path, class_name)
│   → 解析 JSON，返回 {图片名: [放置区域列表]}
│
├── load_debris_images(debris_dir, target_size)
│   → 加载所有异物 PNG（保留 alpha 通道）
│
├── random_transform(debris_img, region, img_size)
│   → 随机缩放 + 旋转 + 在区域内随机定位
│
├── paste_debris(base_img, debris_img, position)
│   → 使用 alpha 通道合成贴图
│
├── compute_iou(box1, box2)
│   → 计算两个框的 IoU（用于避免重叠）
│
├── generate_yolo_label(regions, debris_boxes, img_size)
│   → 生成 YOLO 格式标签文本
│
└── augment_dataset(args)
    → 主流程：遍历 split，解析 JSON，批量扩充
```

## 5. 使用流程

### 5.1 准备阶段

1. **Label Studio 标注：** 对 track_intrusion 和 carriage_rim_debris 的图片标注放置区域
2. **导出 JSON：** 每个 split 的 JSON 放到对应 `labels/` 目录
3. **收集素材：** 异物 PNG 放到 `raw_data/debris_materials/` 各子目录

### 5.2 运行扩充

```bash
# 轨道异物侵限
python scripts/augment_debris.py \
  --dataset_dir "raw_data/track_intrusion" \
  --debris_dir  "raw_data/debris_materials" \
  --class_name  "track_region"

# 车厢沿异物
python scripts/augment_debris.py \
  --dataset_dir "raw_data/carriage_rim_debris" \
  --debris_dir  "raw_data/debris_materials" \
  --class_name  "carriage_rim_region"
```

### 5.3 检查结果

脚本运行后会：
- 打印扩充统计信息
- 生成 `augment_summary.txt` 汇总文件

### 5.4 后续步骤

扩充完成后的数据集可直接用于 YOLO 训练，dataset.yaml 配置：

```yaml
nc: 2
names: ['region', 'debris']
```

---

## 6. 注意事项

1. **异物素材质量：** PNG 必须有透明背景（alpha 通道），否则贴图会有白色/黑色方块
2. **素材多样性：** 每类异物准备 10-20 张不同角度/大小的素材，避免过拟合
3. **扩充比例：** 建议 aug_ratio 设为 0.5~0.8，过多重复扩充可能导致过拟合
4. **放置区域精度：** 标注的区域越精确，贴图越自然
5. **test 集不扩充：** test split 保持原始数据，不做扩充，确保评估客观性
